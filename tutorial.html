对话操作

打开 Claude Code 面板Cmd+I
发送消息Cmd+Enter
清除对话Cmd+K
撤销上次操作Cmd+Z
撤销上一步 Claude 操作Cmd+[

编辑操作

接受建议Tab
拒绝建议Esc
保存文件Cmd+S
切换 diff 视图Cmd+D

导航操作

聚焦输入框Cmd+L
切换侧边栏Cmd+B
打开文件Cmd+P
全局搜索Cmd+Shift+F

终端操作

打开终端Ctrl+`
中断运行Ctrl+C
新终端窗口Cmd+Shift+`

常见问题 FAQ

使用 Claude Code 过程中遇到的问题与解答

可以尝试以下方法：

  • 确保项目根目录有清晰的 README
  • 使用 #file: 显式引用关键文件
  • 运行 claude index 重建项目索引

Claude Code 支持操作撤销：

  • 使用快捷键 Cmd+Z 撤销上一次更改
  • 在对话中输入 撤销刚才的修改
  • 或输入 恢复到上一个版本
问题解决方案
上下文溢出分模块处理，每次专注一个目录
响应缓慢使用 .claudeignore 排除非必要文件
结果不准确提供更精确的文件路径和上下文
  • 不在对话中暴露 API Key 和密码
  • 使用环境变量管理敏感信息
  • 审核 Claude Code 建议的代码变更
  • 配置权限控制限制工具使用范围

在项目根目录维护 .claude/settings.json，通过 Git 分享给团队成员。配置示例：

{
  "language": "zh-CN",
  "customInstructions": "遵循团队的编码规范",
  "hooks": {
    "pre-commit": ["npm run lint:fix"]
  }
}

支持。你可以配置 Hook 或描述定时需求让 Claude 执行：

# 例：每天早上9点检查依赖更新
> 每天 9:00 检查 package.json 依赖版本

表达式说明
0 9 * * *每天早上9点
*/30 * * * *每30分钟
0 0 * * 1每周一凌晨
