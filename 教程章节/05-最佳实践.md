# Claude Code 最佳实践

## 效率提升技巧

### 日常开发流

```
早上: 查看 Issue → Claude 分析 → 确定优先顺序
上午: 编码实现 → Claude 代码审查 → 修正
下午: Claude 生成测试 → 运行验证 → 提交
晚上: Claude 生成 CHANGELOG → 总结当日
```

### Prompt 编写技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| 具体化 | 明确任务范围和约束 | `给 src/auth.ts 添加 JWT 验证中间件` |
| 分步骤 | 复杂任务拆解表达 | `先分析当前代码，再提出优化方案，最后实施修改` |
| 给上下文 | 提供足够的背景信息 | `#file:package.json 中依赖版本是否过时？` |
| 指定格式 | 明确输出格式要求 | `以表格形式列出所有 API 端点` |

## 项目管理

### 代码审查流程

1. **提交前**：Claude 自动审查变更代码
2. **审查关注点**：
   - 逻辑正确性
   - 边界条件处理
   - 安全隐患
   - 代码风格一致性
3. **审查结论**：确认/需修改/拒绝

### 文档维护

- 使用 Claude 自动生成代码注释和 API 文档
- 保持 README 与代码同步更新
- 利用 CHANGELOG.md 记录版本变更

## 团队协作

### 统一配置

```json
// .claude/settings.json（项目级）
{
  "language": "zh-CN",
  "customInstructions": "遵循团队的 TypeScript 编码规范",
  "hooks": {
    "pre-commit": ["npm run lint:fix"]
  }
}
```

### 代码规范强制执行

- 在项目根目录维护 `.claude/settings.json`
- 通过 Hook 实现提交前自动检查
- 团队成员共享同一套配置文件

## 常见问题解答

### Claude Code 无法理解我的项目结构？

```bash
# 确保项目根目录有清晰的 README
# 使用 #file: 显式引用关键文件
# 或运行 claude index 重建项目索引
```

### 如何从错误中恢复？

```bash
# Claude Code 支持操作撤销
Cmd+Z  # 撤销上一次更改
> 撤销刚才的修改
> 恢复到上一个版本
```

### 处理大型代码库

| 问题 | 解决方案 |
|------|---------|
| 上下文溢出 | 分模块处理，每次专注一个目录 |
| 响应缓慢 | 使用 `.claudeignore` 排除非必要文件 |
| 结果不准确 | 提供更精确的文件路径和上下文 |
