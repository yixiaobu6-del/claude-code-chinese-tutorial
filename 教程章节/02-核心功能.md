# Claude Code 核心功能详解

## 对话模式

### 自由对话
最基础的模式，可以问任何编程问题：
```
解释 React 的 useEffect 和 useLayoutEffect 的区别
```

### 上下文对话
引用项目中的文件进行针对性讨论：
```
#file:src/auth.ts 中的登录逻辑有安全漏洞吗？
```

### 多文件对话
同时引用多个文件进行分析：
```
比较 #file:src/v1/api.ts 和 #file:src/v2/api.ts 的差异
```

## 代码编辑

| 操作 | 示例 | 说明 |
|------|------|------|
| 修改函数 | `优化 utils.ts 中的 formatDate 函数` | 精准定位 |
| 添加功能 | `在 user.ts 中添加密码重置功能` | 扩展现有代码 |
| 重构 | `将这个类重构为函数式组件` | 架构调整 |
| 修复 Bug | `修复登录页面的 CSRF 漏洞` | 安全修复 |

## 文件操作能力

```bash
# 创建目录结构
> 创建 src/components/Button.tsx

# 批量重命名
> 将所有 .js 文件重命名为 .ts

# 搜索文件内容
> 搜索所有文件中包含 "apiKey" 的行
```

## 项目管理

### 初始化项目
```
创建一个 React + TypeScript 项目，包含登录注册功能
```
Claude Code 会自动：
1. 初始化 package.json
2. 安装依赖
3. 创建目录结构
4. 生成基础代码

### 依赖管理
```
查看 package.json 中是否有过期的依赖
```

## 调试支持

```
# 查看编译错误
> 运行 tsc 并帮我修复所有类型错误

# 测试失败分析
> 运行测试并分析失败原因
```

## 版本控制集成

```bash
# 查看当前变更
> git diff 查看我的改动

# 生成提交信息
> 根据暂存区的改动生成 commit message
```
