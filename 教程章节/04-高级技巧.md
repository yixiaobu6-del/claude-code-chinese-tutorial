## 自动化工作流

### Hook 系统

Claude Code 支持在特定事件触发时自动执行脚本：

| Hook | 触发时机 | 用途 |
|------|---------|------|
| `pre-tool` | 工具调用前 | 权限校验、日志记录 |
| `post-tool` | 工具调用后 | 结果验证、通知 |
| `pre-commit` | 提交前 | 代码检查、测试运行 |
| `post-commit` | 提交后 | 部署触发 |

### 配置示例

```json
// .claude/settings.json
{
  "hooks": {
    "pre-commit": ["npm run lint", "npm test"],
    "post-commit": ["npm run build"]
  }
}
```

## Cron 定时任务

### 计划任务配置

```bash
# 每天早上9点检查依赖更新
> 每天 9:00 检查 package.json 依赖版本
```

### 常用 Cron 模式

| 表达式 | 说明 |
|--------|------|
| `0 9 * * *` | 每天早上9点 |
| `*/30 * * * *` | 每30分钟 |
| `0 0 * * 1` | 每周一凌晨 |

## 自定义 Skill 高级开发

### 多文件 Skill 结构

```
my-advanced-skill/
├── SKILL.md            # 核心定义
├── instructions/       # 指令文件
│   ├── system.md       # 系统指令
│   └── user.md         # 用户指令
├── tools/              # 自定义工具
│   └── my-tool.ts
└── assets/             # 资源文件
    ├── examples.md     # 示例
    └── templates/      # 输出模板
```

### Skill 链式调用

```yaml
# SKILL.md 中使用 chain 调用其他 Skill
chain:
  - skill: code-review
    input: ${file}
  - skill: test-generator
    input: ${output}
```

## 性能优化

### Token 管理

- 使用 `#file:path` 精确引用文件，避免全量读取
- 大型项目启用 `.claudeignore` 排除非必要文件
- 分步骤处理大型任务，避免上下文溢出

### 上下文窗口策略

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| 分段处理 | 大型重构 | 拆分为多个独立任务 |
| 增量式 | 持续开发 | 每次聚焦一个模块 |
| 聚焦模式 | Bug修复 | 只加载相关文件 |

## 安全最佳实践

- 不在对话中暴露 API Key 和密码
- 使用环境变量管理敏感信息
- 审核 Claude Code 建议的代码变更
- 配置权限控制限制工具使用范围
